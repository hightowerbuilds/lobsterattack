## Deployment / config notes
- **"Supabase not configured" banner** isn't a code miss — Vite inlines `import.meta.env.VITE_*` at dev-server startup / build time and does not re-read `.env` live. A dev server started before `.env` was set shows stale (empty) config → fix is to restart `bun run dev`. `.env` itself is correct (URL + publishable key for `gbnizxzurmbzeelacztr`).
- **Vercel deploy** needs the env vars set in Project → Settings: build-time `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY` (+ `VITE_SUPABASE_ANON_KEY` same value); runtime secret `SUPABASE_SERVICE_ROLE_KEY`. Never `VITE_`-prefix the service-role key (would leak into the client bundle). Do not add `SUPABASE_PASSWORD` (CLI/migrations only). Redeploy after — `VITE_` vars bake in at build time.
